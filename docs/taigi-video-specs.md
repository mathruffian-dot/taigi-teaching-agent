# 本土語教學影片製作規範 (taigi-video-specs.md)

本文件制定了專供 AI Agent 自動化編譯與生成「本土語教學影片」的硬性規格與視覺系統設計。本規範改編自通用教學影片規範，專為本土語（臺灣台語、客語等）的認讀、發音與會話教學進行適配。

---

## 1. 核心定位與三元素

*   **用途**：將一堂課的「核心詞彙」與「情境對話」，自動化編譯為 1–3 分鐘的教學影片（MP4），幫助學生在短時間內掌握發音、漢字與拼音。
*   **典型場景**：國小、國中本土語（閩南語/客語）課堂教學影片。
*   **本土語核心三元素**：
    1.  **漢羅對照字卡**：教育部推薦漢字（Hanji）與台羅拼音（Tailo diacritic）清晰對照，解決「會說不會寫、會寫不會讀」的痛點。
    2.  **生圖插圖 (Option B)**：利用免費 AI 生圖為每個詞彙配上具象化的教學插圖，加強認知聯想。
    3.  **真實發音 (Moedict/TTS)**：讀取本機已下載的教育部萌典官方音訊或雅婷 TTS 語音，保證語音的準確與地道。

---

## 2. 視覺設計系統 (Visual System)

### 2.1 字體與字型
*   **主要字型**：**Noto Sans CJK TC** 或 **源石黑體 (GenSekiGothic2TW)**。
    > [!IMPORTANT]
    > 本土語字型必須支援教育部推薦漢字（如「𨑨迌」、「佇」、「偌」等），若使用不支援的字型會導致文字缺字或變為新細明體。
*   **字重配置**：
    *   漢字標題、巨大強調：**Heavy (900)** 或 **Bold (700)**
    *   台羅拼音、例句：**Medium (500)**
    *   華語翻譯、註解：**Regular (400)**

### 2.2 配色方案 (Color Palette)
本視覺系統採用深墨綠色與米黃色的高對比配置，既符合鄉土自然的意象，又利於投影機或平板閱讀。

| 色碼 | 名稱 | 用途 |
| :--- | :--- | :--- |
| `#0B3C30` | **forest (深墨綠)** | 影片開場/結語背景、文字主色（淺色頁面） |
| `#FAF7EE` | **paper (米黃)** | 內容頁背景色（護眼且高對比） |
| `#E36414` | **coral (強調橘)** | 台羅拼音、關鍵變調字、第二強調色 |
| `#C8941F` | **gold (黃金色)** | 邊框、點綴、複習標記 |
| `#826252` | **mud (泥土棕)** | 輔助文字（如華語對照翻譯） |

## 3. 版面配置與分鏡 (Layout & Storyboard)

影片主要由三部分組成（三明治結構）：

### 3.1 封面與封底 (Dark Theme)
*   **背景**：`#0B3C30` (深墨綠)
*   **排版**：置中。
    *   課名（120px，米黃色，粗體）
    *   台羅課名（70px，強調橘色）
    *   年級與單元（36px，黃金色）

### 3.2 詞彙教學頁 (Left Image, Right Text)
每一頁教一個詞彙，版面採左右分欄（填滿畫布 80% 以上）：
*   **左半部 (45% 寬)**：詞彙插圖。
    *   使用 AI 自動生成的插圖。
    *   必須加上圓角 `16px`、`4px` 的金黃色 `#C8941F` 實線邊框，並帶有輕微陰影。
*   **右半部 (55% 寬)**：文字認讀區。
    *   教育部推薦漢字（130px，深墨綠色，粗體）
    *   台羅拼音（72px，強調橘色）
    *   華語對照（48px，泥土棕色）
*   **下方區塊**：例句。
    *   例句漢字（44px，深墨綠色）
    *   例句台羅（32px，強調橘色）

### 3.3 情境對話頁 (Conversation)
*   **排版**：以「發言角色」與「對話框」呈現。
*   **動畫高亮**：
    *   播放當前句子的音檔時，該對話框高亮（文字顏色為深墨綠，背景為純白卡片），其餘已播放或未播放的對話框透明度降為 `0.4`。
    *   對話框同時呈現：漢字、台羅拼音、華語翻譯。

---

## 4. 動畫節奏與控制 (CSS Animations)

*   **切頁過渡**：採用 CSS `opacity` 漸顯 (0.4s)，避免生硬閃爍。
*   **步驟揭露**：
    *   第 0.0s：頁面載入，顯示左側插圖。
    *   第 0.3s：右側漢字與拼音漸顯，同時觸發音檔播放。
    *   第 0.8s：華語翻譯淡入。
    *   音檔播放結束後，靜止 `2.0s` 緩衝時間供學生朗讀，隨後切換至下一頁。

## 5. 自動化影片渲染管線 (Rendering Pipeline)

```
┌─────────────────────────┐     ┌─────────────────────────┐
│  lesson_structure.json  │ ──> │     generate_html()     │
└─────────────────────────┘     └─────────────────────────┘
                                             │
                                             ▼
┌─────────────────────────┐     ┌─────────────────────────┐
│     Concatenate Audios  │ ──> │     video_slides.html   │
│  (Get durations & sync) │     └─────────────────────────┘
└─────────────────────────┘                  │
             │                               ▼
             │                  ┌─────────────────────────┐
             │                  │    Playwright Record    │
             │                  │    (Headless WebM)      │
             │                  └─────────────────────────┘
             │                               │
             └──────────────> 載入 <─────────┘
                              │
                              ▼
                 ┌─────────────────────────┐
                 │     FFmpeg Mux MP4      │ ──> final.mp4
                 └─────────────────────────┘
```

### 5.1 音訊串接與對齊
*   讀取 `lesson_structure.json` 中的 `audio_file`。
*   使用 `ffmpeg` 的 `concat` filter 或者是實體拼接，生成主音軌 `master_audio.mp3`。
*   利用 `ffprobe` 自動讀取每個音檔的精確秒數，並在 HTML 中產生一個時間陣列（例如 `const PAGE_DURATIONS = [6.2, 5.8, 8.4, ...]`），確保網頁切頁點與音訊播放完美對齊。

### 5.2 瀏覽器錄影 (Playwright)
*   為了解決 Chrome 對音訊「自動播放限制」的阻擋，錄影指令將載入參數 `?render=true`。
*   在 `render=true` 模式下，網頁會自動隱藏點擊遮罩，並藉由錄影腳本的觸發自動開始播放與計時。
*   為避免 Google Drive 的同步鎖定導致 Node.js crash，所有 `node_modules` 與錄影暫存檔一律在 `%TEMP%/taigi-video-render/` 中建立與執行。

### 5.3 最終 mux 合併
*   合併指令必須指定對應的音畫流，避免 Playwright WebM 空音軌覆蓋語音：
    ```bash
    ffmpeg -y -i video.webm -i master_audio.mp3 -map 0:v:0 -map 1:a:0 -c:v libx264 -crf 20 -pix_fmt yuv420p -c:a aac -b:a 192k -shortest final.mp4
    ```
